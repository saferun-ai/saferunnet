# Saferunnet Phase 3 — Stub Crate Activation & Runtime Wiring Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (- [ ]) syntax for tracking.

**Goal:** Activate the three stub crates (exit, platform, rpc), select and integrate an async runtime, replace the StubLokiResolver with a DHT-backed implementation, build the session coordinator module, wire config-driven bootstrap routers into the path builder, and harden service-registration rollback.

**Architecture:** New crates follow the composition-first pattern: traits at the boundary, no deep inheritance. The async runtime (tokio) integrates as a RuntimeHandle service in the kernel. The session coordinator is a new RuntimeModule in saferunnet-app that drives the link session lifecycle through the existing AuthenticatedLinkMessage dispatcher. Exit and platform crates define traits only (no transport implementation yet). RPC crate uses a simple JSON approach.

**Tech Stack:** Rust stable 2024 edition, tokio (async), thiserror, tracing, ed25519-dalek, serde_json (rpc only)
